# testsprite_tests/

This folder contains all test cases and reports generated by **TestSprite MCP** via SpriteStack.

## Structure

```
testsprite_tests/
├── round1/         # Round 1 results (initial test run — found bugs)
│   ├── run-meta-*.json        # Run metadata (status, counts, duration)
│   ├── test-cases-*.json      # Generated test cases with assertions
│   └── report-round1-*.html   # Beautiful HTML report
└── round2/         # Round 2 results (after fixing Round 1 issues)
    ├── run-meta-*.json
    ├── test-cases-*.json
    └── report-round2-*.html
```

## How to Regenerate

1. Start your dev server
2. Run: `spritestack run --round 1`
3. Fix issues from the HTML report
4. Run: `spritestack run --round 2 --compare`

> This folder must be committed to your GitHub repo to qualify for the TestSprite Hackathon.
